FEvER In-App Guide

Preview

Once FEvER reads in the datafile, a preview is provided for quick visual investigation of the input data (useful to figure out if the datafile was parsed correctly) and annotation of the columns for the data analysis.

FEvER uses an internal, in-memory DB to carry out all protein-pathway association queries. Use Select DB button to open DB selection form.

It is possible to view and edit the analysis parameters, using the Parameters button, which will show/hide the parameters form.

Annotation

The data preview table shows the parsed data. Above each column is a drop-down/combo box which contains the possible values for each column. By default all columns are ignored. In order to start an analysis, three sets values are needed: protein identifications, comparative expression values and statistical significance of these values.

  1. Protein identification: Use ProteinID annotation. These are assumed to be UniProt accessions. If multiple IDs are given for a particular row of data (e.g. protein groups, see: protein inference problem) use the Prot ID separation combo-box to select the correct separation character.

  2. Expression values: Multiple options are possible for expression value annotation:

    • Ratio annotation refers to ratio of intensities for the same protein between two samples. These values should be in the range $$$(0, \infty)$$$, where a ratio of $$$1$$$ means no change between the two samples.
    • Fold change annotation is similar to Ratio with the only difference being that the ratios are transformed to be on the same scale. Fold change values have a bimodal distribution, where values in range $$$(-\infty, -1)$$$ denote down-regulation and values in range $$$(1, \infty)$$$ denote up-regulation.
    • Intensity_S1 and Intensity_S2 annotations can be used if the datafile contains multiple intensity values for each sample, originating from replicates. Given that there are at least two measurements per sample, FEvER will automatically calculate the p-values when using this option, using a Student's t-test.

  3. Expression significance: Use Pval annotation. These values reflect the significance of the expression values, calculated using a statistical test. Unless multiple measurements for intensities in both samples are not explicitly annotated, this annotation is required for analysis to continue.
